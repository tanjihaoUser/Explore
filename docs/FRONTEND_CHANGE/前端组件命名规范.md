# 前端组件命名规范文档

## 一、背景框/容器组件的常见叫法

在前端开发中，用于包裹内容、提供背景和布局的容器组件有多种常见叫法：

### 1. 通用容器术语
- **Container（容器）**：最通用的术语，用于包裹页面或组件的主要内容
- **Wrapper（包装器）**：用于包裹其他组件的容器
- **Panel（面板）**：带有背景和边框的内容区域
- **Card（卡片）**：Material Design风格的内容容器，通常有阴影和圆角
- **Box（盒子）**：简单的容器元素
- **Frame（框架）**：用于定义布局结构的容器

### 2. 特定用途容器
- **Dialog/Modal（对话框/模态框）**：用于弹出式内容
- **Drawer（抽屉）**：侧边滑出的容器
- **Sheet（表单页）**：用于表单内容的容器
- **Layout（布局）**：用于页面整体布局结构

## 三、当前项目中的容器命名模式

### 1. 页面主容器命名模式

```css
/* 模式：{页面功能}-container */
.home-container          /* 首页容器 */
.create-post-container   /* 创建帖子容器 */
.post-detail-container   /* 帖子详情容器 */
.simple-page-container   /* 简单页面容器 */
.not-found-container     /* 404页面容器 */
```

### 2. 登录/注册页面特殊命名

```css
/* 背景容器（最外层，包含背景图） */
.login_main             /* 登录/注册页面的背景容器 */

/* 表单容器/背景框（用户看到的表单框） */
.login_form             /* 登录/注册表单容器，这是用户说的"背景框" */
```

### 3. 内容区域命名模式

```css
/* 模式：{功能}-container 或 {功能}-card */
.posts-container        /* 帖子列表容器 */
.post-card              /* 帖子卡片 */
.comments-card          /* 评论卡片 */
.comments-list          /* 评论列表 */
```

### 4. 头部/导航命名模式

```css
.header_main            /* 主头部容器 */
.header-actions         /* 头部操作区域 */
```

### 5. 内容块命名模式

```css
/* 模式：{内容类型}-{位置/功能} */
.post-header            /* 帖子头部 */
.post-content         /* 帖子内容 */
.post-footer            /* 帖子底部 */
.post-author            /* 帖子作者 */
.post-title             /* 帖子标题 */
.post-body              /* 帖子正文 */
```

## 四、Element Plus 组件使用

项目中大量使用了 Element Plus 组件库，这些组件本身也提供了容器功能：

| Element Plus 组件 | 用途 | 对应概念 |
|------------------|------|---------|
| `el-card` | 卡片容器 | Card/Panel |
| `el-dialog` | 对话框 | Dialog/Modal |
| `el-form` | 表单容器 | Form Container |
| `el-container` | 布局容器 | Layout Container |
| `el-scrollbar` | 滚动容器 | Scroll Container |

## 五、命名规范建议

### 1. 容器类命名原则
- 使用 **kebab-case**（短横线命名）
- 语义化命名，清晰表达用途
- 主容器通常以 `-container` 结尾
- 功能区域以功能名开头，如 `post-`, `comment-`, `user-`

### 2. 背景框/面板命名建议
- **Panel（面板）**：`.{功能}-panel`
- **Card（卡片）**：`.{功能}-card`
- **Container（容器）**：`.{功能}-container`
- **Wrapper（包装器）**：`.{功能}-wrapper`
- **Box（盒子）**：`.{功能}-box`

### 3. 当前项目中的"背景框"定义

在登录/注册页面中：
- **`.login_main`**：最外层背景容器（包含背景图片）
- **`.login_form`**：表单背景框（用户看到的浅蓝色表单框，这是用户所说的"背景框"）

## 六、总结

### 背景框的常见叫法（按使用频率）
1. **Panel（面板）** - 最常用，特别是带背景和边框的容器
2. **Card（卡片）** - Material Design 风格，常用于内容展示
3. **Container（容器）** - 通用术语，用于布局容器
4. **Wrapper（包装器）** - 用于包裹其他组件
5. **Box（盒子）** - 简单容器

### 当前项目组件结构
- **页面组件（Views）**：完整的页面视图，使用 `-container` 命名
- **布局组件（Layout）**：页面头部、侧边栏等，使用功能名命名
- **业务组件（Business）**：文章、评论、用户等业务相关组件
- **UI组件库（Element Plus）**：使用 `el-` 前缀的组件

### 登录/注册页面的"背景框"
- 在代码中对应 **`.login_form`** 类
- 这是用户看到的浅蓝色表单容器
- 包含标题、输入框、按钮等表单元素
- 当前尺寸：高度 50%，宽度 30%（第一版大小）
